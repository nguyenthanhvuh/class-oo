Consider a partial version of Bloch's mutable Stack class: 

public class Stack {

   private Object[] elements;
   private int size = 0;

   public Stack() { this.elements = new Object[0]; }

   public void push (Object e) {
     ensureCapacity();      // implementation omitted from quiz
     elements[size++] = e;
   }

   public Object pop () {
     if (size == 0) throw new IllegalStateException("Stack.pop");
     Object result = elements[--size];
     elements[size] = null;
     return result;
   }
}
Also consider the following client interaction with Stack 
        Stack s = new Stack();
        s.push("cat");
        s.push("dog");
        System.out.println(s.pop());
  1. What methods would be needed to convert the client interataction into an equivalent interaction with an immutable version called IStack? For each method, give just the method header - not the contract or implemetation.








    Answer: 
            public IStack push (Object o)
        public IStack pop ()
        public Object top ()
    Note: Obviously, you need a constructor as well, but the simplest thing is for it to behave as the original, so grading is the same whether you list it or not.
  2. Use these methods to write the equivalent interaction with IStack. (You may only use one variable, and its final value must be the same as the value of "s" in the mutable interaction.)








    Answer: 
            Stack s = new Stack();
        s = s.push("cat");
        s = s.push("dog");
        System.out.println(s.top());
        s = s.pop();
  3. How many IStack objects does the interaction produce? (Assume a naive implementation that does not cache IStack objects.)
    Answer: 4. The empty IStack, the "cat" IStack, the "dog" on top of "cat" IStack, and finally the "cat" IStack.
